## Milestones

### v0.3.0 — Network Intelligence

Map networks, not just hosts. Traceroute reveals topology, DNS exposes hidden infrastructure, HTTP enumeration finds attack surface. The topology graph ties it all together visually.

| Feature | Description |
|---------|-------------|
| **Traceroute & hop mapping** | TTL-based hop discovery using raw ICMP. Record each hop's IP, RTT, and hostname. Support both ICMP and TCP traceroute modes. Integrate hop data into HostResult for output. |
| **DNS enumeration** | Zone transfer attempts (AXFR) against discovered DNS servers. Subdomain brute-force using a built-in wordlist. Reverse DNS sweep across subnets. Results feed into host discovery. |
| **HTTP path discovery** | Directory/path brute-force against HTTP services on open ports. Built-in wordlist of common paths (/admin, /api, /.git, /wp-admin, etc). Report discovered paths with status codes. |
| **Network topology graph** | Extend HTML report with an interactive network diagram. Use traceroute hop data to draw network layers. Show hosts as nodes with open ports, group by subnet. SVG or canvas-based rendering. |

### v0.4.0 — Advanced Evasion

Outsmart firewalls and IDS. Understand how defenders detect scanners, then build countermeasures. Each feature here teaches you about a real detection technique and how to bypass it.

| Feature | Description |
|---------|-------------|
| **Source port control** | `--source-port <port>` flag. Use common ports like 53 (DNS) or 80 (HTTP) as source port to bypass stateless firewall rules that allow established connections from these services. |
| **Adaptive timing** | Monitor response rate and packet loss during scans. Auto-slow when detecting drops (possible rate limiting). Auto-speed-up when target is responsive. Replace fixed timing templates with intelligent pacing. |
| **Rate limiting detection** | Detect when a target starts dropping packets or responding slower. Log warnings when rate limiting is suspected. Auto-backoff to avoid triggering IDS alerts. |
| **Proxy/SOCKS support** | Route connect scans through SOCKS4/5 or HTTP proxies. `--proxy socks5://host:port` flag. Enables scanning from different source IPs and through network boundaries. Only applies to connect/service detection (not raw scans). |

### v0.5.0 — Protocol Depth

Go deeper into protocol internals. Build scanners for protocols most tools ignore. Each one requires understanding packet structures at the byte level.

| Feature | Description |
|---------|-------------|
| **IPv6 raw scan support** | Extend raw scan engine to support IPv6. Build ICMPv6 neighbor discovery, TCP SYN over IPv6 using raw sockets. Handle IPv6 extension headers. Currently all IPv6 targets fall back to connect scan. |
| **SCTP INIT scanning** | Send SCTP INIT chunks and analyze responses. INIT-ACK = open, ABORT = closed. Used in telecom (SS7, Diameter) and VoIP (SIP over SCTP). Requires building SCTP packets from scratch. |
| **QUIC/HTTP3 probing** | Send QUIC Initial packets (UDP-based) to detect HTTP/3 services. Parse version negotiation responses. Increasingly common on modern web infrastructure (Google, Cloudflare, etc). |
| **SNMP walking** | Send SNMPv1/v2c GetNext requests to walk MIB trees on open port 161. Extract system description, interface list, routing tables. Common on network devices, printers, IoT. |

### v0.6.0 — Extensibility & Automation

Make nmapper programmable. A scripting engine turns it from a tool into a platform. Alerting enables continuous security monitoring.

| Feature | Description |
|---------|-------------|
| **Scripting engine** | Embed Rhai (Rust-native) or Lua runtime. Users write scripts that define custom probes: send bytes, parse responses, report findings. Similar to Nmap's NSE but in a Rust-native tool. Scripts loaded from `~/.nmapper/scripts/`. |
| **Webhook/Slack alerting** | Send scan results and watch-mode alerts to webhooks. `--webhook <url>` flag. JSON payload with host/port changes. Enables integration with Slack, Discord, PagerDuty, or custom dashboards. |
| **XML/Nmap-compatible output** | `-o xml` output format matching Nmap's DTD. Enables importing nmapper results into Metasploit, OpenVAS, and other tools that consume Nmap XML. |

### v0.7.0 — Advanced Techniques & Platform

Exotic scan techniques and cross-platform support. Idle scan is one of the most clever techniques in network security. Platform expansion makes nmapper usable everywhere.

| Feature | Description |
|---------|-------------|
| **Idle/zombie scan** | Use a third-party "zombie" host's IP ID sequence to scan a target without sending any packets from your own IP. Requires finding a host with predictable IP IDs (incremental). The most stealthy scan technique that exists. |
| **ARM cross-compilation** | CI pipeline and build scripts for ARM targets (Raspberry Pi, embedded devices). Enables deploying nmapper as a portable network probe on small devices. |
| **Plugin system** | Loadable plugin architecture. Plugins define new scan types, service probes, or output formats. Dynamic loading via Rust's `libloading` or compiled-in via feature flags. |
| **Windows support** | Replace macOS BPF backend with Npcap/WinPcap on Windows. Abstract the datalink layer behind a platform trait. Conditional compilation with `#[cfg(target_os)]`. |
